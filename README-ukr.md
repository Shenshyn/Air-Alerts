# Аналіз виживання (Survival Analysis) повітряних тривог

Цей проект використовує методи аналізу виживання (survival analysis) для дослідження даних про повітряні тривоги в Україні. Головна мета — прогнозування тривалості тривог та розуміння факторів ризику (таких як регіон, час доби та одночасність тривог в інших областях).
В проекті використовуються базові оцінки Каплана-Мейєра (Kaplan-Meier), моделі пропорційних ризиків Кокса (Cox Proportional Hazards) та моделі виживання XGBoost (XGBoost Survival).

## Огляд

Основний набір даних містить історичні записи повітряних тривог для різних регіонів України. Завдання полягає у моделюванні тривалості цих тривог, де завершення (відбій) тривоги розглядається як "подія" (event) у контексті аналізу виживання.

## Основні проблеми та їх вирішення

Під час розробки та аналізу ми зіткнулися з кількома критичними проблемами в даних, які вимагали спеціального підходу:

### 1. Масове дублювання даних
**Проблема**: Аудит сирих даних (`official_data_en.csv`) показав, що приблизно 41.66% рядків були точними копіями. Це сталося через повторне зчитування блоків краулером під час початкового збору даних.
**Рішення**: Застосовано сувору глобальну дедуплікацію (`df.drop_duplicates()`) як перший крок у пайплайні обробки даних, щоб уникнути зміщення при навчанні моделей.

### 2. Зміна методології збору та зміщення оцінки (Evaluation Bias)
**Проблема**: Наприкінці 2025 року методологія збору даних змінилася: почали фіксувати тривоги на рівні районів/громад. Як наслідок, дані за 2026 рік мали сильне зміщення в бік міста Київ (який залишився єдиною зоною рівня області). Простий хронологічний спліт для тестування (наприклад, тестова вибірка = дані 2026 року) призводив до того, що тестовий набір складався майже виключно з Києва, що унеможливлювало адекватну оцінку передбачуваної здатності для всієї країни.
**Рішення**: Ми відмовилися від простого хронологічного поділу на користь **Стратифікованого хронологічного спліту (Stratified Chronological Split)**. Ми виконали хронологічний поділ 80/20 *незалежно для кожного регіону*. Це гарантує, що тестова вибірка містить останні дані, які пропорційно репрезентують всю країну. Ми також відфільтрували дані, залишивши лише стратегічні тривоги рівня області (`level == 'oblast'`), щоб уникнути шуму від прифронтової артилерії.

### 3. Колінеарність та сингулярні матриці в моделі Кокса
**Проблема**: При навчанні моделі пропорційних ризиків Кокса виникала помилка `LinAlgError: Matrix is singular`.
**Рішення**: Причиною став екстремальний дисбаланс класів: для Луганської області в наборі даних було лише 2 події. Включення регіонів з майже нульовою кількістю подій створює сингулярну матрицю при оцінці часткової правдоподібності моделі Кокса. Ми відфільтрували Луганську область та Крим (регіони без регулярних даних про тривоги) для стабілізації моделі. Також до моделі Кокса було додано невеликий L2-пеналізатор (penalizer) для запобігання подальшим проблемам з колінеарністю.

### 4. Артефакти "наївного" цензурування (Naive Censoring)
**Проблема**: Для деяких активних тривог на момент експорту даних час завершення (`ended_at`) був наївно встановлений на дату експорту (наприклад, червень 2026 року), що створювало штучно гігантські тривалості.
**Рішення**: Ми суворо вираховували тривалість від `started_at` до `finished_at`. Будь-які нереалістичні тривалості або тривоги без підтвердженого `finished_at` оброблялися належним чином (або фільтрувалися, щоб не спотворювати базову лінію).

## Методологія

### Конструювання ознак (Feature Engineering)
- **Конкурентність (Concurrency)**: Кількість інших активних тривог рівня області в той самий момент, коли почалася поточна тривога. Це було ефективно реалізовано за час $O(N \log N)$ за допомогою алгоритму скануючої прямої (sweep-line).
- **Часові ознаки**: Циклічне кодування (перетворення через синус і косинус) години, дня тижня та місяця для врахування сезонності та ефектів часу доби.
- **Просторові ознаки**: One-hot кодування областей для встановлення регіональних базових рівнів ризику.

### Моделі
1. **Базова модель Каплана-Мейєра**: Забезпечує непараметричну оцінку ймовірностей виживання для кожного регіону.
2. **Модель пропорційних ризиків Кокса**: Напівпараметрична модель для оцінки впливу наших сконструйованих ознак (співвідношення ризиків - hazard ratios).
3. **Модель виживання XGBoost**: Підхід на основі градієнтного бустингу дерев, оптимізований за допомогою цільової функції `survival:cox`, для захоплення нелінійних взаємодій.

## Результати моделей

Моделі оцінювалися за допомогою Індексу конкордації (C-index), який вимірює здатність моделі правильно ранжувати час виживання.

| Модель | Тестовий C-index |
| --- | --- |
| **XGBoost (survival:cox)** | ~ 0.6524 |
| **Cox Proportional Hazards** | ~ 0.6382 |
| **Kaplan-Meier (Regional Baseline)** | ~ 0.62 |

*Примітка: C-index 0.5 означає випадкове вгадування, а 1.0 — ідеальне ранжування. З огляду на дуже стохастичну природу повітряних тривог, C-index ~0.65 вказує на сильний прогностичний сигнал у порівнянні з базовою лінією.*

## Структура проекту
- `src/data_loader.py`: Очищує дані, видаляє дублікати, фільтрує області, розраховує тривалість та кешує оброблені дані.
- `src/baseline_km.py`: Будує криві виживання Каплана-Мейєра для ключових регіонів.
- `src/model_cox.py`: Навчає модель Cox PH та виділяє важливість ознак.
- `src/model_xgb.py`: Навчає модель виживання XGBoost.
- `src/evaluate.py`: Порівнює моделі за допомогою стратифікованого хронологічного спліту та генерує графіки продуктивності.
- `plots/`: Містить згенеровані візуалізації (криві виживання, порівняння моделей).

## Запуск коду

1. Переконайтеся, що встановлені необхідні бібліотеки: `pandas`, `numpy`, `lifelines`, `xgboost`, `matplotlib`, `seaborn`.

3. Запустіть пайплайн оцінки: `python3 src/evaluate.py`
